Getting started

In this tutorial, we will show how to use this web site effectively.

If you click on the yellow highlighted Macaulay2 code, the results of running those commands will be displayed in the right hand window. You may also type directly into the right hand window.

For example, click on this yellow code to compute the factors of the given polynomial.

R = QQ[x,y,z]

F = (x-1)^3 - (y-1)^3

factor F

This code is evaluated by Macaulay2 on the server. This means that you don't need to have Macaulay2 or any of its included packages installed on your computer.

You are on the first page of this tutorial. At the top of the window, you see how many pages are in this tutorial. After you are finished with a page, click or tap on the right arrow key. You can also go back to previous pages of the tutorial by clicking on the left arrow button.

Right now, please continue to the next page of the tutorial.

Getting Macaulay2 help on a topic

There are two main ways of getting help on a topic in Macaulay2. The first method gives you text help in the Macaulay2 output window. For example,

help determinant

This displays in the Macaulay2 output window how to use the function "determinant". Note! In many browsers, you will have scroll bars to go back, but in some, notably most versions of Safari, the scrollbars are there but hidden!

Alternatively, and more usefully, you may obtain the web version by using:

viewHelp determinant

This opens the documentation page in another browser window or tab. You can surf through the links there to see other Macaulay2 documentation.

Depending on your browser settings, you may be navigated to this new page, or not. Your Macaulay2 web session is still active in any case! You might need to disable popup window blocking in your browser in order to get the help page.

Using the terminal and the Input window

You can type in Macaulay2 commands directly into the right hand side terminal. The usual shell like behavior works, such as up and down arrows and tab completion.

Using the Input window on the left

When you write M2 functions or long loops, it is often more convenient to write your code in a separate window and execute the code after you wrote it. This allows you to easily modify previous code and move the cursor around freely.

Clicking or tapping the Input button at the top of the page, navigates you to the Input Window. This is where you may type in Macaulay2 code and have it evaluated.

In that window, place your cursor on a line that you wish to evaluate, and either click the Evaluate button, or press Shift-Return (both keys at the same time).

Alternatively, you may select several lines, or a part of a line, and clicking Evaluate (or press Shift-Return) will evaluate the selection.

You try it! Click Input, type in some Macaulay2 input, or try out some of the example lines already there. When you are ready to come back to the tutorial, click or tap on the Tutorial button at the top of the page. You will end up right here!

Macaulay2 sessions and the Reset button

When your browser starts, it starts a Macaulay2 process on our server. All of your input, either typed and evaluated, or run by pressing buttons, goes to this process. If you open a second window or tab in the same browser, you will get the same session.

You might wish to start fresh, or restart Macaulay2, if a computation is going on too long. In this case, press the Reset button at the top of the page.

When you first connect to this web site, a cookie is generated, and that is used to connect to your Macaulay2 process on the server. If you quit your browser, and it is set to remember windows, then it remembers the cookie, and you can reconnect back to the same session. If not, then a new session is created. The server will remove the session after some length of time, currently several days or a week.

Sometimes the server must be restarted for maintenance or other reasons, and at those times, all sessions are terminated.

Downloading your work

Click on the Save button at the top right of the window. A dialog will appear where you can save the text in the Input window, or the text in the Macaulay2 terminal on the right-hand side. Alternatively, you can just select the text you wish, and copy it.

Advanced topics

Uploading files

Use the Upload File button to upload files to the Macaulay2 server that can then be loaded into your Macaulay2 session. Suppose that you have uploaded a file named foo.m2 which contains Macaulay2 code. You may then load this file into Macaulay2 using either:

load "foo.m2"

or, if you wish to see all of the lines executed one by one:

input "foo.m2"

Note: these lines will likely give errors now, as the file foo.m2 probably doesn't exist on the server!

Running shell commands in your Macaulay2 session

It is possible to run shell commands from within Macaulay2, although many operations are restricted. For example, to see what files you have in your home directory on the server, try:

get "!ls -a"

Macaulay2 and image files

Sometimes Macaulay2 or one of its packages generates image files. For example, the displayGraph function below creates a jpeg file.

needsPackage "Graphs"

A = graph({{v_1,v_3},{v_2,v_4},{v_1,v_4}})

displayGraph A

Getting files back to your machine

Sometimes your Macaulay2 code will create a file as output. How to download this onto your machine? Here is one way. This first line creates a file for the purpose of this example.

"important-results.txt" << "very important results" << close

The next line will present a download dialog for you to obtain the file on your machine.

get "!open important-results.txt"

Restrictions in place for the Macaulay2 server

The server is being shared with other users, therefore memory and CPU time are limited (generous, but limited). Other restrictions include: upload file size, number of processes that can run, and the amount of time that the session is saved on the server.
